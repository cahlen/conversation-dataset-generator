About The Project

Ever wish you could generate just the right kind of conversational data? Whether you're fine-tuning a Large Language Model (LLM) for a specific style or persona, need dialogue for a creative project, or want to explore complex topics in a natural flow, the Conversation Dataset Generator is here to help!

This powerful and flexible Python script leverages Hugging Face's transformers library to put you in control, operating in two main modes:

  • Manual Mode: Specify everything ‚Äì the exact topic, personas, scenario, style, and keywords.
  • Creative Brief Mode: Provide a high-level creative-brief and let the script brainstorm details, generate variations, and optionally use web search for richer context.

Either way, the output is a clean JSON Lines (.jsonl) file, perfect for downstream tasks and compatible with popular LLM training frameworks.

Output Format Sample (JSON Lines)
{ "conversation_id": 0, "turn_number": 0, "role": "human", "speaker_name": "Alex", "topic": "the absurdity of everyday errands", "scenario": "waiting in line at the post office", "style": "observational, witty", "include_points": "long lines, confusing forms", "content": "Seriously, Sam, look at this line..." }

Why Use This Generator?

Style Specialization: Train models to master specific conversational nuances.

Persona Embodiment: Build believable characters, even niche ones (using web search).

Topic Fluency: Enhance model ability to discuss particular subjects naturally.

Instruction Adherence: Train models to better follow constraints (e.g., keywords).

Creative Content Generation: Break writer's block and draft dialogue.

Dialogue Flow Analysis: Study conversation progression via structured output.

Getting Started

Prerequisites

  • Python 3.8+
  • GPU (Recommended, especially for Brief Mode)
  • Capable CPU & Adequate RAM
  • Internet Connection (for --personaX-search-term)
  • Install Dependencies (see Installation)

Installation

# 1. Clone the repo (Optional) git clone https://github.com/cahlen/conversation-dataset-generator.git cd conversation-dataset-generator # 2. Create & Activate Virtual Env (Recommended) python3 -m venv venv source venv/bin/activate # Win: venv\Scripts\activate # 3. Install Base Dependencies pip install -r requirements.txt # 4. Install Specific PyTorch (Optional for GPU) # See https://pytorch.org/get-started/locally/ # Example: pip3 install torch ... --index-url ... # 5. Install Optional Dependencies pip install duckduckgo-search # For Brief Mode web search pip install -U peft trl bitsandbytes # For LoRA examples # 6. Login to Hugging Face Hub (Optional) huggingface-cli login

Usage

Modes of Operation

Manual Mode

Provide all parameters explicitly.

python generate.py \ --persona1 "Wizard" --persona1-desc "Grumpy" \ --persona2 "Knight" --persona2-desc "Cheerful" \ --topic "polishing armor" \ --scenario "dungeon waiting room" \ --style "Comedic" \ --num-examples 10

Brief Mode

Provide a high-level brief; use LLM + optional web search.

python generate.py \ --creative-brief "Pirate orders coffee" \ --num-examples 15 \ --persona1-search-term "Pirate captain traits"

Script Flow Overview

1. Start generate.py 2. Check arguments: |--> If --delete-repo provided: | Confirm deletion -> Delete repo(s) -> Exit. |--> If --creative-brief provided: | Optional: Perform web search (if --personaX-search-term) | Generate base args (personas, topic, etc.) using LLM (+ web context) | Optional: Search for persona images | Loop N times (--num-examples): | Generate topic/scenario variation using LLM | Generate conversation using LLM (base personas + varied context) | Parse & Store Conversation | -> Save to JSONL -> Optional Upload -> Exit. |--> Else (Manual Mode): | Use provided --topic, --personas, etc. | Optional: Search for persona images | Loop N times (--num-examples): | Generate conversation using LLM (fixed context) | Parse & Store Conversation | -> Save to JSONL -> Optional Upload -> Exit.

Argument Reference

Key arguments to control generation (provide EITHER --creative-brief OR manual args):

--creative-brief: High-level concept (Brief Mode).
--personaX-search-term: Web search query (Brief Mode, Optional).
--topic: Conversation subject (Manual Mode).
--scenario: Conversation setting (Manual Mode).
--style: Dialogue style (Manual Mode).
--personaX: Speaker name (Manual Mode).
--personaX-desc: Speaker description (Manual Mode).
--include-points: Keywords to include (Manual Mode, Optional).
--num-examples: Number of conversations.
--output-file: Output .jsonl path.
--model-id: Hugging Face generation model.
--upload-to-hub: HF Hub repo ID (Optional).
--delete-repo: DANGER: Deletes HF repo.

See the full argument reference in the README for details and defaults.

Output Format

Data is saved locally as a .jsonl file (one JSON object per line/turn). Optionally, it can be uploaded to the Hugging Face Hub as a structured Dataset.

Each turn includes: conversation_id, turn_number, role, speaker_name, topic, scenario, style, include_points, and content.

Output Format Sample (JSON Lines)
{ "conversation_id": 0, "turn_number": 0, "role": "human", "speaker_name": "Alex", "topic": "the absurdity of everyday errands", "scenario": "waiting in line at the post office", "style": "observational, witty", "include_points": "long lines, confusing forms", "content": "Seriously, Sam, look at this line..." } { "conversation_id": 0, "turn_number": 1, "role": "gpt", "speaker_name": "Sam", "topic": "the absurdity of everyday errands", "scenario": "waiting in line at the post office", "style": "observational, witty", "include_points": "long lines, confusing forms", "content": "Only if the vortex requires triplicate forms..." }

See the full output format details in the README.

Model & Fine-Tuning Notes

  • Uses meta-llama/Meta-Llama-3-8B-Instruct by default (change with --model-id).
  • Generated data is ideal for PEFT methods like **LoRA** for style/persona/topic specialization.
  • LoRA benefits: smaller footprint, faster training, modular, shareable.
  • For best fine-tuning results, use strong instruction-following base models (Llama 3, Mistral, Mixtral, Qwen2, Gemma Instruct, etc.).
  • Training LoRAs requires additional libraries: peft, trl, bitsandbytes.
